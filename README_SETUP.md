# 英単語シャッター ver 1.7：開発者画面表示安定化版

## URL構成

- 生徒用アプリ：`/`
- 開発者用アプリ：`/developer.html`
- 開発者用画面の「生徒用アプリを確認」から、画面内プレビューを表示可能

開発者用アプリは、URLを開けばそのまま使用できます。パスワード設定やログイン操作はありません。

## 今回の主な機能

- アップロードする単語データごとに、開発者側で「学習メニュー名」を設定
- 設定した名前とサムネイルを、生徒用アプリの教材選択画面へ表示
- 複数の教材を生徒用アプリへ追加し、サムネイル付きカードから選択可能
- 各Unitを20語ごとのPartに自動分割
- 生徒側では、学習メニュー → Unit → Partの順で選択
- 同じUnit・Partの学習を途中まで終えた場合、次回は未出題の続きから出題
- 順番通り・シャッフルのどちらでも、Part内の単語を一巡するまで重複を避ける
- 同じ学習メニュー名を指定すると、既存メニューへ単語を追加・更新、または内容を置き換え可能
- 将来の本棚形式へ変更しやすいよう、学習メニューごとにデータを分けて保存

## 初回設定

### 1. GitHubへ配置してVercelへデプロイ

このフォルダ内のファイルを、Vercelと連携しているGitHubリポジトリのルートへ配置してください。

`package.json` と `api` フォルダがあるため、Vercelが必要なパッケージとAPIを準備します。

### 2. Vercel Blobを作成

1. Vercelで対象プロジェクトを開く
2. `Storage` を開く
3. `Blob` のストアを **Public** で作成し、対象プロジェクトへ接続
4. `BLOB_READ_WRITE_TOKEN` がプロジェクトの環境変数へ追加されたことを確認
5. 必要に応じて再デプロイ

### 3. 学習メニューを追加

1. `https://あなたのURL/developer.html` を開く
2. 生徒側に表示する学習メニュー名を入力
3. 単語ファイルをアップロード
4. 必要に応じて、その下の欄から教材サムネイルをアップロード
5. 読み取りプレビューを確認
6. `生徒用アプリへ反映する` を押す

反映後、生徒用アプリを再読み込みすると、設定した名前とサムネイルが教材選択画面へ表示されます。サムネイルを設定しない教材は「No Image」と表示されます。

## 添付する単語ファイル

添付ファイルには、原則として **英単語と意味だけ** を入れてください。

- 1列目：英単語
- 2列目：意味
- 1行目に「英単語」「意味」という見出しを入れても構いません
- 対応形式：Excel、CSV、TSV、TXT、JSON、ODS、PDFなど
- PDFは文字を選択できる形式に対応。画像だけのPDFは読み取り対象外
- 最大ファイルサイズ：4MB
- Excel・ODSで `元` シートがある場合は優先して読み取り、なければ先頭シートを読み取る
- TXTは「英単語と意味」をタブ・カンマ・縦線のいずれかで区切る
- JSONは、`[{"word":"apple","meaning":"りんご"}]` の形式、または2列の配列形式に対応
- `【動】`、`【名】` などの品詞ラベルごとに、複数選択の正解候補へ分割

余計な情報を誤って読み取る可能性を減らすため、別の表・説明文・成績情報などは同じファイルに入れないでください。

## ZIP内のファイル名

文字化けを避けるため、ZIP内のフォルダ名とファイル名はすべて半角英数字に統一しています。

- ルートフォルダ：`english_vocab_shatte_ver_1.6`
- テンプレート：`vocabulary_template.csv`、`vocabulary_template.txt`、`vocabulary_template.json`

## Unit・Partの分割

- 1 Unit：最大100語
- 1 Part：最大20語
- 例：Unit1の1〜20語目がPart1、21〜40語目がPart2
- 最後のUnit・Partは、登録語数に合わせて20語未満になる場合があります

## 学習メニュー名と反映方法

### 新しい名前を入力した場合

新しい学習メニューとして生徒用アプリへ追加されます。

### すでに存在する名前を入力した場合

- **現在の単語に追加・更新する**
  - 同じ英単語：意味を更新
  - 新しい英単語：末尾へ追加
- **アップロード内容で置き換える**
  - その名前の学習メニューを、今回アップロードした内容だけで作り直す

## データ保持

- 公開された学習メニュー・単語データ：Vercel Blobに保存
- 生徒の学習履歴・Partの続き位置：各生徒端末のブラウザ内に保存
- アプリのコードを更新しても、同じドメイン・同じブラウザを使う限り学習履歴を引き継ぐ
- 同じ英単語の学習履歴は、学習メニューをまたいでも英単語名を基準に引き継ぐ
- ブラウザのサイトデータ削除、別端末、別ドメインへの変更では学習履歴を引き継げない

## 公開範囲について

開発者用アプリにはログイン制限がありません。`/developer.html` のURLを知っている人は、学習メニューと単語データを変更できます。生徒へURLを共有しない運用を前提としています。

## 将来の本棚画面への拡張

各教材は、名前・ID・単語データ・サムネイルを分けて保存しています。教材選択画面はカード形式で、今後の本棚形式へ拡張しやすい構成です。


## ver 1.5 の変更点

- 「学習の仕方」を「学習メニュー」の横へ移動
- 出題順・1回に学習する数・制限時間を「学習の仕方」に集約
- 「通常学習」「苦手な単語」「間違えた単語」を選択可能
- 定着具合の「あと少し」を「うろ覚え」に変更
- 通常学習では、これまでどおり同じUnit・Partの続きから出題
- 苦手な単語・間違えた単語の学習では、選択中のPartから対象単語を抽出


## ver 1.6 の変更点

- 生徒用アプリに「教材を選択」画面を追加
- 教材数が増えても探しやすいよう、サムネイルカードと教材名検索を追加
- 開発者用アプリで、単語ファイルの下からサムネイル画像を追加・変更可能
- サムネイル未設定時は「No Image」を表示
- 同じ教材名へ反映するとき、画像を選択しなければ既存サムネイルを保持
- 「サムネイルを外す」でNo Imageへ変更可能
- サムネイル画像はブラウザ側で横長に切り抜き・軽量化し、Vercel Blobへ保存
- これまでの教材データ・学習履歴は引き続き読み込み可能


## ver 1.7 の変更点
